# Bio-AI 科研情报雷达 - 2026-05-10

## 一、Top 高星区技术风向

近期高星 Bio-AI 项目呈现以下三大技术趋势：

1. **LLM Agent 化全面渗透生物信息学**：高星项目中，Agent 类项目占比极高。从 `bioagent`（端到端自主研究）到 `bioinformatics-agent-skills`（MCP 服务器封装 78 个生信工作流），再到 `celltype-agent`（单细胞注释）和 `variant-triage`（临床变异解读），Bio-AI 正从“工具调用”向“自主推理 + 工具编排”的 Agent 范式快速演进。**LangGraph 成为主流编排框架**，多个项目（bioagent、agentic-genomics）明确使用。

2. **工具化与平台化并重，MCP 协议成为新基建**：`bioinformatics-agent-skills` 和 `AgentFigureGallery` 均采用 MCP（Model Context Protocol）或类似“技能”机制，将生信工具封装为 AI 编码 Agent 可调用的标准化接口。这标志着 Bio-AI 工具正从“独立脚本”向“Agent 可插拔技能”转型，**MCP 生态正在成为生物信息学 Agent 的基础设施层**。

3. **单细胞组学工具链加速 Rust/Python 重写**：`rustscenic`（SCENIC+ 的 Rust 重写）、`cnvturbo`（inferCNV 的 Python 重写，~100× 加速）、`scnoisemeter`（单细胞噪声量化）等项目表明，社区正在用现代语言和架构重构经典生信工具，追求**性能提升、依赖简化、生态整合**（如与 Scanpy/AnnData 深度绑定）。同时，`vcfkit` 和 `poa-consensus` 等 Rust 工具也体现了这一趋势。

**判断**：当前 Bio-AI 高星项目更偏 **LLM Agent 化** 和 **工具平台化**，多模态化尚不明显（仅 `celltype-agent` 涉及空间转录组学多模态数据）。作物基因组学领域出现 `OneGenome-Rice` 这样的基因组基础模型，但星数尚低，趋势待观察。

## 二、最新更新区低星潜力项目深挖

在最新更新区中，以下低星项目具有较高跟踪价值：

### 值得继续跟踪的项目清单

| 项目 | 理由 |
|------|------|
| **narrow-model-safety-eval** | 明确涉及蛋白质语言模型（ESM-2）和结构设计工具（ProteinMPNN）的双重用途风险评估，提供 HuggingFace 数据集，有论文/评估框架线索。这是 AI 生物安全的前沿方向。 |
| **llm-evaluation-framework** | 医学 QA 领域的 LoRA 微调评估框架，有量化结果（+17.8% ROUGE），可能对应可复现的实验代码。 |
| **agentic-runtime-platform** | 生产级多 Agent 编排平台，虽非直接 Bio-AI，但其 DAG 执行、模型路由、评估框架可被生信 Agent 系统复用。 |

### 详细分析

**narrow-model-safety-eval**（⭐ 0，更新于 2026-05-10）
- **线索**：README 明确提及 ESM-2 和 ProteinMPNN，提供 HuggingFace 数据集（`narrow-model-safety-eval`），有 CI 和 CC BY 4.0 许可证。
- **判断**：这是一个**可复现的双重用途风险评估实验代码库**，可能对应预印本或论文。值得关注其是否发布 bioRxiv 论文。
- **建议**：跟踪其 HuggingFace 数据集和后续论文发布。

**llm-evaluation-framework**（⭐ 0，更新于 2026-05-10）
- **线索**：README 提供具体评估结果（Base Mistral 7B vs LoRA 微调模型，+17.8% ROUGE，65% win rate），涵盖 5 个临床领域。
- **判断**：这是一个**可复现的医学 LLM 评估实验**，但未提及论文或模型权重发布。需要进一步核验是否对应已发表研究。
- **建议**：跟踪其是否发布模型 checkpoint 或扩展评估。

**agentic-runtime-platform**（⭐ 0，更新于 2026-05-09）
- **线索**：README 描述生产级多 Agent 编排，支持 8+ LLM 提供商，有 MkDocs 文档。
- **判断**：这是一个**通用 Agent 基础设施项目**，虽非直接 Bio-AI，但其架构可被生信 Agent 系统采用。无论文线索。
- **建议**：作为基础设施层跟踪，关注其是否被生信项目引用。
