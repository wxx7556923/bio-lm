# Bio-AI 科研情报雷达 - 2026-05-25

## 一、Top 高星区技术风向

近期高星 Bio-AI 项目呈现出以下显著趋势：

1.  **基因组基础模型进入“工业化”阶段**：以 Hugging Face 的 **Carbon** 为代表，基因组基础模型正从学术研究走向平台化、工程化。Carbon 不仅提供了在 1T tokens/6T DNA 碱基对上训练的大模型，更重要的是配套了标准化的评估代码（零样本变异效应预测、序列恢复等），旨在解决当前 DNA 评估生态碎片化、可复现性差的问题。这表明该领域正在从“造模型”转向“建生态”。

2.  **AI Agent 与生物信息学深度融合，走向“工具化”与“Agent 化”**：
    -   **AgentFigureGallery** 和 **awesome-skills** 这类项目，将科学绘图、AI4Protein 等生物信息学技能打包成可供 Claude Code、Codex 等编码 Agent 直接调用的“技能包”，实现了生物信息学工作流的 Agent 化、低代码化。
    -   **GeneX-AI** 则更进一步，构建了一个集成 LLM 多智能体推理与确定性物理/生态求解器的通用计算生物学框架，其内部的多智能体辩论（架构师→理论物理学家→评审委员会→实验学家）流程，代表了将 LLM Agent 用于复杂科学问题拆解与验证的前沿探索。

3.  **性能优化向底层语言（Rust）和专用硬件加速倾斜**：**scvelo-rs**（Rust + PyO3 加速单细胞 RNA 速率）、**gffbase**（Rust + DuckDB 加速 GFF3/GTF 解析）和 **COSMolKit**（Rust 原生化学信息学工具包）等项目，不约而同地选择 Rust 语言来重写或加速生物信息学核心流程，追求极致的性能和内存安全。这反映了生物信息学工具开发正从“能用”向“高效、可扩展”演进。

4.  **空间组学与微生物组学成为新热点**：**UniSOR** 专注于空间分辨多组学和显微镜数据的高保真恢复，**Genos-m** 则发布了针对人类微生物组的基因组基础模型。这表明基因组基础模型的应用正在从单一的“人/模式物种”基因组，向更复杂的空间维度和更广泛的微生物群落扩展。

**趋势判断**：当前 Bio-AI 领域正从“模型创新”驱动，转向“平台化、工具化、Agent 化”的生态建设阶段。核心趋势是：**用 LLM Agent 封装复杂分析流程，用 Rust 等高性能语言优化底层基础设施，用基础模型统一处理多模态、多尺度的生物数据。**

## 二、最新更新区低星潜力项目深挖

在“最新更新”区中，我们重点关注了 stars 较低但更新活跃的项目，发现了以下值得跟踪的潜力项目：

### 值得继续跟踪的项目清单

1.  **PACE** (`AI4Science-WestlakeU/PACE`)
    -   **理由**：这是一个来自西湖大学 AI4Science 实验室的几何感知桥接传输方法，用于从非配对、破坏性的单细胞时间序列快照中恢复连续的细胞发育轨迹。项目明确关联了 **arXiv 论文 (2605.18587)**，并提供了核心实现、配置和示例数据。这是一个典型的“新论文+可复现代码”项目，对于单细胞轨迹推断领域具有重要价值。

2.  **CatIF-RL** (`lynk-101-li/CatIF-RL`)
    -   **理由**：该项目专注于“活性导向的酶序列设计”，通过强化学习引导的逆向蛋白质折叠来实现。项目明确关联了 **bioRxiv 论文 (10.64898/2026.05.11.724288)** 和 **Zenodo 数据集 (10.5281/zenodo.20357062)**，提供了完整的代码、数据和模型检查点线索。这是 AI4Protein 领域一个非常具体且有潜力的应用方向。

3.  **KaryoScope** (`barthel-lab/KaryoScope`)
    -   **理由**：该项目提出了一种快速、无需比对的序列注释方法，专为泛基因组时代设计。项目明确关联了 **bioRxiv 论文 (2026.05.15.725544)**。在泛基因组日益重要的背景下，这种无需比对的方法有望解决传统比对方法在大型、复杂泛基因组上的计算瓶颈。

4.  **Genos-m** (`BGI-HangzhouAI/Genos-m`)
    -   **理由**：由华大基因（BGI）杭州AI团队发布的人类微生物组基因组基础模型。项目提供了 **Hugging Face 模型权重** 和一份 **技术报告 (PDF)**。虽然 stars 不高，但作为国内头部基因测序公司发布的微生物组基础模型，其技术路线和性能值得密切关注。

5.  **scvelo-rs** (`ilaykav/scvelo-rs`)
    -   **理由**：这是一个用 Rust + PyO3 对 scVelo 进行加速的项目，旨在作为 scVelo 的“即插即用”加速替代品。项目已发布到 **PyPI**，并提供了完整的 CI/CD 流程。对于需要处理大规模单细胞 RNA 速率分析的用户来说，这是一个非常实用的性能优化工具。

6.  **UniSOR** (`LinShuhaiLAB/UniSOR`)
    -   **理由**：一个用于空间分辨多组学和显微镜数据高保真恢复的统一框架。虽然 readme 中未直接提及论文，但其描述非常具体，涉及多种前沿技术（MSI, SRT, SRP），且项目结构完整。这是一个高度专业化的工具，值得空间组学领域的研究者关注。
